Данный проект содержит класс SharedRes с общим для всех потоков свойством типа string.
Для синхронизации доступа к свойству используется класс ReaderWriterLockSlim из пространства имен System.Threading.

Разница между Debug и Trace заключается в том, что все методы в Debug вызываются только тогда, когда установлен DEBUG символ компиляции 
(по умолчанию включен для отладки, выключен для realease), когда символ не установлен, методы никогда не вызываются в вашем коде. 
Trace ищет символ TRACE (по умолчанию включен как для отладки, так и для выпуска). Кроме этого, два класса идентичны.

Для реализации множественного доступа к разделяемому ресурсу используется класс Task, который для выполнения задачи берет поток из пула потоков
и может переиспользовать их. Это эффективнее, чем создание нового потока для каждой новой задачи.
